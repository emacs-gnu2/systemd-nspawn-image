* main
** Introduction
This repository stores the image files to run some systemd-nspawn containers, therefore you can import image via http from here.

For all image files, see [[https://github.com/emacs-gnu2/systemd-nspawn-image/releases/][release page]]

For usage introduction and other information, see [[https://codeberg.org/gituser/systemd-nspawn-config][another configurations repository]] on codeberg.org.
** Currently supported containers
*** No prepare.sh support
+ =container-trixie-mini=, Minimal installation of debian13
*** With prepare.sh support
+ =pubmail=, transform from docker-mailserver project, use mysql to store email account info.
+ =container-prod-jenkins=, run a jenkins server
+ =container-prod-openresty=, run a nginx server (based on openresty)
+ =container-sandbox-browser=, run firefox via waypipe
